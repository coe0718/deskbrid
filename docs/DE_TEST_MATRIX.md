## Architecture

- тЬЕ actions on all DEs are **backend-independent** тАФ handled via D-Bus, sysfs, systemd, AT-SPI2, CDP, or OS processes
- Core DE-specific actions use the `DesktopBackend` trait тАФ all 9 backends override 44 mandatory methods
- `windows.tile` composites `system_info()` + `window_move_resize()` тАФ move_resize gaps cascade to tile
- `windows.activate_or_launch` composites `windows_list()` + `window_focus()` + daemon spawn
- `layout_profiles.save/restore` are daemon-level orchestrations
