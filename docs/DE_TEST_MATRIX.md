## Architecture Notes

- тЬЕ actions marked on *all* DEs are **backend-independent** тАФ handled in the daemon layer via D-Bus (NetworkManager, BlueZ, PipeWire, logind, upower, geoclue), sysfs (backlight, thermal, CPU), systemd (services, journal, timers), AT-SPI2 (accessibility), Chrome DevTools Protocol (browser), or pure OS processes (process, terminal, files).
- Core DE-specific actions (windows.*, workspaces.*, input.*, clipboard.*, screenshot, notifications, monitor.*) go through the `DesktopBackend` trait тАФ all 9 backends override the 44 mandatory trait methods.
- `windows.tile` is a daemon-level composite of `system_info()` + `window_move_resize()`, so gaps in move_resize cascade to tile.
- `windows.activate_or_launch` composites `windows_list()` + `window_focus()` + daemon process spawn, so it works even on DEs without move/resize.
- `layout_profiles.save/restore` are daemon-level orchestrations compositing multiple backend trait calls.
