# The Problem

Linux has no unified desktop.

Mac gives you one AX API. Windows has UIA. Linux has nine compositors that all think they're special. Each with its own protocol, its own quirks, its own way of breaking.

AI agents need to click buttons, read screens, type keystrokes. Without Deskbrid, that means reimplementing the same intent nine different ways — and keeping them at parity as each compositor drifts.

# The Solution

Write once.
Run everywhere.

Deskbrid auto-detects your desktop environment at startup. GNOME, KDE, Hyprland, Sway, COSMIC, Niri, Wayfire, Labwc, or X11 — one protocol, one daemon, one API. The agent doesn't need to know what it's running on. Deskbrid handles the translation.

$ deskbrid --demo

It speaks agent.

# Unix socket — agent-native
echo '{"type":"windows.list","id":"1"}' |
  nc -U $XDG_RUNTIME_DIR/deskbrid.sock

# CLI — human-native
$ deskbrid windows list
→ Code — OSS [1920×1080]
→ Terminal — zsh [1920×1080 *]
→ Firefox — github.com [2560×1440]
# Python client
from deskbrid import Deskbrid

client = Deskbrid()
client.focus_window(app_id='code')
client.type_text("Hello from AI!▊")
client.screenshot(save='desktop.png')

# Capabilities

Everything an agent
needs to drive a desktop.

🪟

Windows & Workspaces

List, focus, close, minimize, maximize, tile, move, resize. Full window management across all compositors.

⌨️

Keyboard & Mouse

Type text, send key combos, control the mouse. Virtual input devices via uinput — works everywhere.

📋

Clipboard

Read, write, and track clipboard history. Cross-compositor pasteboard access.

📸

Screenshots & OCR

Capture screens, diff regions, extract text. Built-in OCR for reading what's on screen.

📊

Live Dashboard

Web UI at localhost:20129 — system info, running windows, clipboard history, audit log, all streaming live via SSE.

🔌

MCP Integration

100+ MCP tools across 22 categories. Claude Desktop, Codex, Cursor — any MCP client. deskbrid mcp
